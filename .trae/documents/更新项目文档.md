# 更新项目文档计划

## 问题分析

我发现项目文档中存在一些与实际代码不符的地方，主要包括：

1. **API文档过时**：API_REFERENCE.md中显示的类构造函数仍然使用多个直接参数，而实际代码现在只接受一个config字典参数
2. **方法签名不正确**：一些方法如`visualize`不存在于VisionPipeline类中
3. **示例代码过时**：README.md和QUICKSTART.md中的示例代码使用了旧的API格式
4. **参数名称不一致**：一些参数名称已经更改，如`class_conf_thresholds`改为`category_thresholds`

## 解决方案

1. **更新API_REFERENCE.md**：
   - 修改VisionPipeline、YOLODetector等类的构造函数文档，改为只接受config字典参数
   - 更新方法签名，移除不存在的方法，添加新方法
   - 更新参数名称，确保与实际代码一致

2. **更新QUICKSTART.md**：
   - 修改示例代码，使用新的config字典参数格式
   - 移除对不存在方法的调用
   - 更新方法调用方式

3. **更新README.md**：
   - 修改示例代码，使用新的API格式
   - 确保文档中的示例与实际代码一致

4. **检查其他文档**：
   - 检查FEATURES.md等其他文档，确保内容与实际代码一致

## 实施步骤

1. 首先更新API_REFERENCE.md，确保API文档与实际代码一致
2. 然后更新QUICKSTART.md，确保快速开始示例能够正常运行
3. 接着更新README.md，确保项目首页的示例代码正确
4. 最后检查其他文档，确保内容一致

## 预期结果

- 所有文档都与实际代码一致
- 示例代码能够正常运行
- API文档准确反映类和方法的实际签名
- 文档能够正确指导用户使用项目
